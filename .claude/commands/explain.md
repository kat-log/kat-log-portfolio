# /explain - 実装振り返りコマンド

直前に実装した内容について、以下の4つの観点で解説してください。

## 1. 実装概要
- 何を作ったのか
- どんな問題を解決したのか
- ファイル構成や主要な変更点

## 2. 動作確認の方法・手順
- 具体的なコマンドや操作手順
- 期待される結果
- 「これが表示されたらOK」という成功基準

## 3. 技術的なポイント
- なぜこの実装方法を選んだのか
- 設計上の工夫や注意点
- 今後の拡張性について

## 4. 用語解説
- 使用した技術・ライブラリの説明
- 初めて出てきた概念の解説
- 関連する学習リソースがあれば紹介

---

特に「動作確認」は丁寧に、コピペで実行できるレベルで書いてください。
コマンドは必ずプロジェクトのパッケージマネージャー `pnpm` を使用してください。
